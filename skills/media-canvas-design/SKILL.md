# Canvas Design Workflow

Treat this file as a router. First define the visual philosophy, then load only the canvas-execution guidance needed to make the artifact.

## Workflow

1. Start with `references/philosophy.md`.
   - Use it to create the design philosophy and deduce the subtle conceptual reference from the user’s request.
2. Only after the philosophy is solid, read `references/canvas-workflow.md`.
   - It covers canvas creation, typography expectations, final-polish rules, and the multi-page extension.
3. When typography matters, inspect `canvas-fonts/` and choose fonts that support the philosophy rather than defaulting to generic system fonts.
4. Deliver the visual artifact plus the philosophy file.

## Rules

- Do not jump straight to composition without first establishing the philosophy.
- Keep the work design-forward and visually led; text stays sparse unless context clearly demands a stronger typographic gesture.
- Prefer refinement and compositional tightening over piling on extra elements.
