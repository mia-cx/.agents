# Doc Co-Authoring Workflow

Treat this file as a router. Keep the live process small in-context and load the deeper stage guidance only when the user is actually in that stage.

## Workflow

1. Offer the structured co-authoring workflow when the user is writing a substantial document.
2. Figure out the current stage:
   - context gathering and doc setup
   - section-by-section drafting and refinement
   - reader testing and final review
3. Read only the matching guide:
   - `references/workflow.md` for context gathering and iterative drafting
   - `references/reader-testing.md` for fresh-reader validation and final review
4. If the user declines the workflow, switch to freeform help instead of forcing the process.

## Rules

- Use the workflow as scaffolding, not bureaucracy.
- Keep stage transitions explicit so the user knows what happens next.
- Do not load reader-testing guidance until the draft is mature enough to test.
