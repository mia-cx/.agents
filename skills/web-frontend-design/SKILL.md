# Web Frontend Design

Treat this file as a router. Read the design-principles guide before coding so the interface direction is intentional rather than generic.

## Workflow

1. Read `references/design-principles.md` for worktree setup, design thinking, aesthetic direction, and execution standards.
2. Commit to a clear visual concept before writing implementation code.
3. Match the implementation complexity to the chosen aesthetic.

## Rules

- Prefer strong point-of-view over generic “nice-looking” UI.
- Keep the main file as a routing layer; the detailed design guidance lives in the reference.
- Use this skill when interface polish matters more than app scaffolding.
