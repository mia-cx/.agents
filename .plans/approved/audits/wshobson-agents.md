# Role-to-SOP audit: wshobson-agents

Compiled from `plans/audits/wshobson-agents/raw-findings.md` per FORMAT-PROMPT. **Meta-context** (`plans/reference-material/theo-meta-context-v2.md`): portable value lives in **procedure** (checks, gates, evidence)—not persona labels.

§9 lists traceable citations; narrative outside table cells labels **interpretation** where the prompt asked for inference.

## 1. Repo overview

**Interpretation.** wshobson-agents is a Claude Code–oriented **plugin marketplace**: `plugins/<domain>/` with `.claude-plugin/plugin.json`, optional `skills/`, `agents/`, and `commands/`. Findings describe procedure-heavy Markdown (Type, Trigger, Steps/contract, Strip, Notes) with recurring bindings to Conductor paths, `~/.claude` team layout, and tool-specific literals.

## 2. Content summary

**From findings shape:** each entry summarizes portability and a workflow. **Interpretation:** skills carry the densest cross-stack SOPs; agents shorten role behaviors; commands chain flows; `plugin.json` rows are distribution metadata without executable steps.

## 3. SOP split

### 3.1 SOPs to port (recommended)

**Interpretation.** Team coordination + review/debug skills; Conductor **patterns** after artifact rename; stack-selected developer-essentials; incident trio (postmortem/handoff/runbook); doc skills (ADR/OpenAPI/changelog); `Portable: Yes` agents as trimmed subagent prompts.

### 3.2 SOPs to leave out (or defer)

**Evidence samples:** `Portable: No` appears on mesh skills (`istio-traffic-management`, `linkerd-patterns`), `spark-optimization`, `plugin-eval` items, several reverse-engineering agents, and more—see raw-findings. **Interpretation:** skip manifests; defer `?` blocks; abstract `Partially` conductor commands before binding to files.

**Evidence vs interpretation:** §4 pulls **Trigger/Steps/Strip/Notes** from findings (condensed); §3 prioritization and §5 bands are judgment.
