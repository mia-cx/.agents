**Source repo**: `.references/agentic-rules/`

---

## 1. Repo overview

`agentic-rules` is a rules and conventions repository published by AINative Studio to govern an AI-agent-driven software development workflow built around a named coding agent called "Cody," an MCP-as-automation-bus harness, GitHub Issues for backlog management, and HyperScaler (AWS Lambda / Azure Functions / GCP Cloud Functions) deployment targets. Its purpose is to give Cody a comprehensive operating ruleset spanning the full SDLC: issue triage, sprint planning, chain-of-thought reasoning before code, TDD, refactoring, CI/CD pipeline generation, canary deploy, rollback/monitoring, and self-validation at task close. The repo went through at least one consolidation: `.references/agentic-rules/globalrules.md` and `.references/agentic-rules/meta-rules.md` were tombstoned and merged into `.references/agentic-rules/canonical-rules.md` as of 2026-03-27. Alongside the rule files, the repo ships a prompt-library companion (`.references/agentic-rules/Agentic-Prompts-Library.md`, `.references/agentic-rules/Enhanced-Agentic-Prompts-Library.md`), a 25-agent persona catalog (`.references/agentic-rules/Agent-Personas.md`), and a `.references/agentic-rules/templates/claude-code/` directory of scaffold files (slash commands, CLAUDE.md, settings JSON, agent skeletons). The project is explicitly scoped to AINative's product and is not designed for portability.

## 2. Content summary

The repository contains five content categories. **Rule files** (`.references/agentic-rules/canonical-rules.md`, `.references/agentic-rules/meta-rules.md`, `.references/agentic-rules/globalrules.md`, `.references/agentic-rules/MVP-GlobalRules.md`, `.references/agentic-rules/agent-reasoning-planning-execution.md`) are long flat Markdown documents with numbered top-level sections covering SDLC phases; they carry no YAML frontmatter and mix procedural steps with inline LLM prompt examples and cloud-provider CLI commands. **Prompt libraries** (`.references/agentic-rules/Agentic-Prompts-Library.md`, `.references/agentic-rules/Enhanced-Agentic-Prompts-Library.md`) catalogue block-quoted MCP invocation templates for Cody; they are companion how-to-invoke artefacts, not standalone SOPs. **The agent persona catalog** (`.references/agentic-rules/Agent-Personas.md`) defines 25 specialist agent roles in a consistent H3-per-role structure plus three collaboration pattern descriptions at the end. **Template scaffolds** (`.references/agentic-rules/templates/claude-code/`) contain fill-in-the-blank files where every field is a `{{PLACEHOLDER}}` token; they define schemas but encode no executable behaviour. **Configuration artefacts** (`.references/agentic-rules/templates/claude-code/settings.json.template`, `.references/agentic-rules/templates/claude-code/.claude/settings.local.json.example`, `.references/agentic-rules/templates/claude-code/scripts/merged-permissions.json`, `.references/agentic-rules/templates/claude-code/claude.json.template`) are JSON payloads demonstrating permission models and MCP server stubs. Only `.references/agentic-rules/strict-rules-how-to.md` and `.references/agentic-rules/templates/claude-code/.claude/commands/review.md` are short, self-contained, and immediately portable without reformatting.

## 8. Structural patterns

### Frontmatter
No file in the repo uses YAML frontmatter. Rule files are pure Markdown with numbered section headers. This means no `description:` trigger field exists for automated skill-loader matching — triggering is entirely manual or harness-driven.

### Section numbering
`.references/agentic-rules/canonical-rules.md` and `.references/agentic-rules/agent-reasoning-planning-execution.md` use §1–§N top-level numbered sections with sub-numbered rules (e.g., `§2.1`, `§2.2`). `.references/agentic-rules/agent-reasoning-planning-execution.md` uses consistent sub-field labels within each rule: **Rule:**, **Format:**, **Output:**, **Structure:**. This pattern is the most systematic in the repo and the closest to a machine-parseable schema.

### Naming conventions
Rule files use `kebab-case` for filenames. Template files use `PascalCase` (e.g., `.references/agentic-rules/templates/claude-code/CLAUDE.md.template`, `.references/agentic-rules/MVP-GlobalRules.md`) or `camelCase` (`.references/agentic-rules/globalrules.md`, `.references/agentic-rules/canonical-rules.md`). No consistent convention enforced across the repo.

### Template convention
All scaffold files use `{{DOUBLE_BRACE_PLACEHOLDER}}` for variable substitution (consistent with `.references/agentic-rules/docs/LLM_PROJECT_SETUP_PROMPT.md`'s stated convention). Slash command files use `$ARGUMENTS` for runtime parameterisation (Claude Code-specific).

### Permission entry format
`.references/agentic-rules/templates/claude-code/scripts/merged-permissions.json` and `.references/agentic-rules/templates/claude-code/.claude/settings.local.json.example` use two entry patterns: `Bash(<subcommand>:*)` for shell tool grants (glob-matched) and `mcp__<server>__<tool>` for MCP tool grants. The `deny` and `ask` arrays in `.references/agentic-rules/templates/claude-code/scripts/merged-permissions.json` are intentionally empty — the repo takes an allowlist-only stance, relying on Claude Code's default-block behaviour for anything not in `allow`.

### Packaging observations
- Rule files are monolithic: all SDLC phases in a single file. Individual sections are self-contained and extractable, but the packaging makes it easy to miss that §3 and §8 are already promoted to live skills.
- The `templates/` directory is a sibling to `.references/`, not nested under a common `artifacts/` or `scaffolds/` directory — no consistent separation between runnable content and structural scaffolding.
- Two files (`.references/agentic-rules/globalrules.md`, `.references/agentic-rules/meta-rules.md`) carry explicit tombstone notices in their content ("SUPERSEDED — DO NOT EDIT") but are not physically removed from the repo. This creates a false audit surface: a reader must read the tombstone notice before knowing the file is dead.
- Prompt library files (`.references/agentic-rules/Agentic-Prompts-Library.md`, `.references/agentic-rules/Enhanced-Agentic-Prompts-Library.md`) are not marked as companion artefacts to `.references/agentic-rules/canonical-rules.md`. Their relationship is only inferable from content overlap, not from any metadata.
